# dbt Project

This project builds a lightweight analytics layer on top of the DuckDB warehouse generated by the Python ETL pipeline.

## Prerequisites

- Python 3.12
- The project virtual environment with dependencies installed

## Setup

From the repository root:

```bash
python3.12 -m venv .venv-dbt
source .venv-dbt/bin/activate
pip install dbt-duckdb
```

## Run the ETL first

The DuckDB warehouse must exist before dbt can build models from it.

```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python - <<'PY'
from pipeline.main import run_pipeline
run_pipeline('data/raw/fmn_data.xlsx', 'data/warehouse.duckdb')
PY
```

## dbt commands

From the dbt project directory:

```bash
cd dbt
source ../.venv-dbt/bin/activate
dbt debug --profiles-dir .
dbt run --profiles-dir .
dbt test --profiles-dir .
```

## What gets built

- Staging models for transactions, products, distributors, salespersons, monthly targets, and the date table
- Two marts:
  - `mart_sales_performance`
  - `mart_distributor_product_summary`

The DuckDB database used by dbt is:

```text
data/warehouse.duckdb
```
